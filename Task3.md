## 1. DDD — доменные зоны

### 1. User & Access Domain (Пользователи и доступ)

**Описание:**
Управляет пользователями, ролями и авторизацией.

**Состав:**

* Регистрация / авторизация
* Роли (HR, сотрудник, админ)
* Профили пользователей

**Глоссарий:**

* **User** — пользователь системы
* **Role** — роль (HR, employee, admin)
* **Session** — сессия пользователя
* **Auth Token** — токен авторизации

---

### 2. Learning Management Domain (Управление обучением)

**Описание:**
Основная логика обучения: курсы, программы, уроки.

**Состав:**

* Курсы и модули
* Программы обучения
* Назначение курсов

**Глоссарий:**

* **Course** — курс
* **Lesson** — урок
* **Program** — набор курсов
* **Assignment** — назначение курса сотруднику

---

### 3. Assessment Domain (Оценка и тестирование)

**Описание:**
Отвечает за тесты и определение уровня сотрудника.

**Состав:**

* Входной тест
* Оценка результатов
* Уровни навыков

**Глоссарий:**

* **Assessment** — тест
* **Question** — вопрос
* **Score** — результат
* **Skill Level** — уровень навыка

---

### 4. Progress & Analytics Domain (Прогресс и аналитика)

**Описание:**
Отслеживает обучение и предоставляет отчеты.

**Состав:**

* Прогресс по курсам
* Completion rate
* Отчеты для HR

**Глоссарий:**

* **Progress** — прогресс пользователя
* **Completion** — завершение курса
* **Report** — отчет
* **Metric** — метрика

---

### 5. Recommendation Domain (Рекомендации)

**Описание:**
Формирует персональные рекомендации.

**Состав:**

* Рекомендации курсов
* Карьерные треки
* AI-логика

**Глоссарий:**

* **Recommendation** — рекомендация
* **Career Track** — карьерный трек
* **Skill Gap** — дефицит навыков

---

## 2. BDD — критический путь MVP

### Критический путь:

Сотрудник проходит входной тест → получает курс → начинает обучение

---

### Сценарий успеха

**Дано:**

* сотрудник зарегистрирован в системе
* сотруднику назначен входной тест

**Когда:**

* сотрудник заходит в систему
* проходит тест и отправляет ответы

**Тогда:**

* система рассчитывает уровень сотрудника
* система назначает стартовый курс
* сотрудник видит курс в личном кабинете

---

### Сценарий отказа

**Дано:**

* сотрудник начал проходить тест

**Когда:**

* сотрудник отправляет тест
* происходит ошибка сервера

**Тогда:**

* система сообщает об ошибке
* ответы пользователя не теряются
* пользователю предлагается повторить попытку

## 4. API-First — JSON-контракты

### 1. POST /assessments/submit

**Описание:** Отправка результатов входного теста

**Request:**

```json
{
  "user_id": "123",
  "assessment_id": "entry_test_1",
  "answers": [
    {
      "question_id": "q1",
      "answer": "A"
    },
    {
      "question_id": "q2",
      "answer": "C"
    }
  ]
}
```

**Response (success):**

```json
{
  "status": "success",
  "score": 78,
  "level": "middle",
  "recommended_course_id": "course_456"
}
```

**Response (error):**

```json
{
  "status": "error",
  "message": "Failed to process assessment"
}
```

### 2. GET /courses/recommendations

**Описание:** Получение рекомендаций курсов

**Request:**

```json
{
  "user_id": "123"
}
```

**Response:**

```json
{
  "user_id": "123",
  "recommendations": [
    {
      "course_id": "course_456",
      "title": "Digital Marketing Basics",
      "level": "middle"
    },
    {
      "course_id": "course_789",
      "title": "Advanced Analytics",
      "level": "advanced"
    }
  ]
}
```
